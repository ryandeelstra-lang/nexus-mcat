# Consensus-verification spec (Nexus knowledge graph — Wave 2)

You are an INDEPENDENT verifier for one MCAT section. The topic layer was generated by other agents; your
job is the "check" half of "create and check": confirm the prerequisite edges are correct, remove wrong
ones, and ADD the high-value cross-domain prerequisites that a single-scope generator couldn't see.

## Inputs (read these)
- `docs/data/mcat_topics.yaml` — the FULL assembled topic layer: every topic/subtopic id + name for the
  whole MCAT, plus the current `topic_prerequisites` (each with a rationale + confidence). This is your
  complete id space — you may reference any id in it.
- Optionally the `docs/data/topics/gen_*.yaml` fragments for extra rationale context.

## Your scope
You own ONE section (given in your prompt): all topic/subtopic ids whose id begins with your section's
prefix (`BB.` = B-B, `CP.` = C-P, `PS.` = P-S, `CARS.` = CARS).

## What to produce
1. **drop** — prerequisite edges whose **destination (`dst`) is in YOUR section** that are WRONG (reversed
   direction, not an actual prerequisite, circular, or nonsensical). Be conservative: only drop genuinely
   incorrect edges; keeping a correct edge is the default (you do NOT need to list edges you keep).
2. **add** — NEW prerequisite edges whose **`dst` is in YOUR section**, especially **cross-domain** links
   coming FROM other sections (e.g. C-P chemistry → B-B biochemistry; B-B physiology → P-S behavior;
   physics → fluids/gas exchange). Use EXACT ids from `mcat_topics.yaml`. These inter-domain edges are the
   most valuable part of the graph — add the ones that are real and missing. Each needs a rationale +
   confidence. `src` may be in any section; `dst` must be in YOURS.
3. **topics_flag** (optional) — ids of topics/subtopics in your section that look wrong, duplicated, or
   mis-scoped, with a one-line issue. (Reported for human review; not auto-applied.)

## Hard rules
- Endpoints must be EXACT ids that exist in `mcat_topics.yaml` (a topic or subtopic id) — never a bare
  content-category id (e.g. never `BB.1A`).
- `dst` of every drop/add must be in YOUR section.
- Do not introduce cycles (the assembler will also enforce acyclicity, but don't rely on it).
- Write ONLY your assigned file; make no other edits.

## Output YAML shape (exactly)
```yaml
section: "B-B"
drop:
  - { src: "BB.1A.T05", dst: "BB.1A.T01", reason: "reversed — amino acids precede enzyme kinetics" }
add:
  - { src: "CP.5E.T01", dst: "BB.1D.T01", rationale: "Thermodynamics underlies bioenergetics", confidence: "high" }
  - { src: "CP.5D.T02", dst: "BB.1A.T01", rationale: "Organic chemistry of functional groups underlies amino acids", confidence: "high" }
topics_flag:
  - { id: "BB.2B.T04", issue: "duplicate of BB.2B.T02" }
```

After writing, reply with a 2-line summary: how many edges you dropped / added / flagged.
